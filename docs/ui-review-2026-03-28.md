## P0-1 全局 Toast 反馈系统

### 模块定义

这不是一个页面，而是**贯穿全应用的反馈基础设施**。所有用户操作的成功/失败/进行中状态，都通过此系统反馈。

### 组件选型

使用 `sonner` 的 `<Toaster />` 组件（shadcn/ui 已封装）。

```tsx
// app-shell.tsx 根组件中注入
import { Toaster } from "@/components/ui/sonner";

// 布局
<>
  <div className="flex h-screen">
    <Sidebar />
    <main className="flex-1"><Outlet /></main>
  </div>
  <Toaster position="top-right" richColors closeButton />
</>
```

### 位置与样式

```
位置：top-right（视口右上角）
偏移：top: 16px, right: 16px
宽度：356px（固定）
堆叠：最多显示 3 条，新消息从上方推入
圆角：rounded-lg (8px)
阴影：shadow-lg
动画：从右侧滑入 200ms ease-out，淡出 150ms
```

### Toast 类型规格

| 类型 | 调用方式 | 图标 | 背景色 | 持续时间 | 可关闭 |
|------|---------|------|--------|---------|--------|
| 成功 | `toast.success(msg)` | ✓ (CheckCircle) | bg-green-50 border-green-200 | 3000ms 自动消失 | ✓ |
| 错误 | `toast.error(msg)` | ✗ (XCircle) | bg-red-50 border-red-200 | 不自动消失 | ✓ 必须手动关闭 |
| 信息 | `toast.info(msg)` | ⓘ (Info) | bg-blue-50 border-blue-200 | 4000ms | ✓ |
| 加载 | `toast.loading(msg)` | ⏳ (Loader2 spin) | bg-muted | 跟随操作 | ✗ |
| Promise | `toast.promise(fn)` | 自动切换 | 自动切换 | 跟随操作 | ✓ |

### 各场景文案模板

```typescript
// === 项目设置 ===
toast.success("项目设置已保存");
toast.error("保存失败：网络错误，请重试");

// === 全局设置 LLM ===
toast.success("LLM 配置已保存");
toast.error("保存失败：请检查网络连接");

// === LLM 连接测试 ===
toast.promise(testConnection(), {
  loading: "正在测试连接...",
  success: (data) => `连接成功 — 模型: ${data.model}，延迟 ${data.duration}ms`,
  error: (err) => `连接失败：${err.message}`,
});

// === API 测试用例生成 ===
toast.promise(generateCases(), {
  loading: "正在生成测试用例...",
  success: (data) => `已生成 ${data.length} 条用例`,
  error: "用例生成失败，请检查 LLM 配置",
});

// === 测试执行 ===
toast.success("执行完成：12 通过 / 2 失败");
toast.error("执行中断：服务器连接超时");

// === OpenClaw 连接 ===
toast.success("连接已添加");
toast.success("连接已更新");
toast.error("连接已删除");  // 用 toast 而非 error，删除是预期操作
toast.info("连接已断开");

// === 表单验证 ===
toast.error("请填写必填字段：名称、网关地址");

// === 存储路径 ===
toast.success("存储路径已更新，立即生效");
toast.warning("请先将旧目录数据迁移至新路径");
```

### 按钮 Loading 状态规范

所有提交按钮在请求期间：

```tsx
<Button disabled={mutation.isPending}>
  {mutation.isPending ? (
    <>
      <Loader2 className="h-4 w-4 animate-spin mr-2" />
      保存中...
    </>
  ) : (
    "保存"
  )}
</Button>
```

规则：
1. `disabled={true}` — 防止重复提交
2. 文案变为「动作+中...」（保存中...、连接中...、生成中...）
3. 图标替换为 `Loader2` 旋转动画
4. 按钮宽度不变（使用 `min-w` 或固定宽度避免跳动）

### 交互流程

**导入文档流程：**
```
用户点击 [导入文档]
  → 跳转到 /p/$projectId/api/import
  → 完成导入后返回 API 测试页
  → 接口列表自动刷新（TanStack Query invalidate）
  → 列表展示导入的接口和模块分组
```

**生成测试用例流程：**
```
用户选中接口（checkbox）→ 点击 [生成选中(N)]
  → 按钮变为 [⏳ 生成中...] disabled
  → toast.loading("正在生成测试用例...")
  → 成功 → toast.success("已生成 X 条用例") + 列表刷新
  → 失败 → toast.error("生成失败：{原因}")
  → 按钮恢复

无选中时按钮文案为 [全部生成]，对所有接口生成
有 tag 筛选时文案为 [生成「tagName」]
```

**执行测试用例流程：**
```
用户点击 [全部执行] / [执行选中(N)]
  → runningBatch = true
  → 工具栏下方出现 Progress 进度条
  → 按钮变为 [⏳ 8/20]
  → 逐条执行，进度条实时更新
  → 完成 → 进度条消失 + toast.success("执行完成：X 通过 / Y 失败")
  → 各用例行更新状态图标（✓ 绿色 / ✗ 红色）
```

**查看用例详情流程：**
```
用户点击用例行
  → 右侧 Sheet 滑出（w-[480px]）
  → 显示用例名称、tag、Monaco 编辑器（request/expected JSON）
  → 底部显示最近一次执行结果（如果有）
  → 用户可编辑 JSON → 点击 [保存] → toast.success("用例已更新")
  → 用户可点击 [▶ 执行] → 按钮 loading → 结果实时显示在底部面板
```

### 全局搜索快捷键

```
Cmd+K / Ctrl+K → 聚焦搜索框（autoFocus）
Escape → 清空搜索并失焦
搜索为实时过滤（onChange），无需回车
支持：接口路径、HTTP 方法、summary 文本
```

### 交互流程

**进入设置页：**
```
用户点击侧边栏 [项目设置]
  → 路由导航到 /p/$projectId/settings
  → TanStack Query 加载 project 数据
  → 加载中：骨架屏
  → 加载完成：表单填充现有值
```

**编辑并保存：**
```
用户修改任意字段
  → 点击 [保存]
  → 按钮变为 [⏳ 保存中...]，disabled
  → 成功 → toast.success("项目设置已保存") + 按钮恢复
  → 失败 → toast.error("保存失败：{原因}") + 按钮恢复
  → Query 缓存自动刷新（项目列表、当前项目）
```

**管理请求头：**
```
点击 [+ 添加]
  → 新增一行空的 key-value 输入
  → 新行自动 focus 到 key 输入框
  → Tab 键跳到 value 输入框
  → 点击 🗑 删除行（无确认，直接移除）
  → 保存时自动过滤空 key 的行
```

**跳转 OpenClaw 管理：**
```
点击 [管理连接]
  → 路由导航到 /p/$projectId/openclaw
  → 侧边栏高亮切换到 OpenClaw
```

### 测试连接交互流程

```

  → 成功 → toast.success("连接成功，延迟 {N}ms")
         → 按钮下方显示 Badge: ✅ 连接正常 · {N}ms · 模型: {model}
```
